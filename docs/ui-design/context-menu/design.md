# 右键上下文菜单 — UI 设计稿

## 1. 设计概述

**设计目标**: 使用 macOS 原生 `NSMenu` 实现右键上下文菜单，零自定义样式，完全遵循系统视觉语言。

**设计原则**:
- **原生优先**: 不做任何自定义渲染，NSMenu 自动适配深色/浅色模式、VoiceOver、高对比度
- **上下文感知**: 根据点击目标显示不同菜单，菜单项数量精简，避免信息过载
- **操作直达**: 菜单项直接映射到操作，无二级子菜单

**与项目视觉语言的关系**: 无自定义视觉元素。NSMenu 自动继承系统主题，与 OWL 工具栏/侧边栏的原生 AppKit 控件保持一致。

## 2. 信息架构

右键菜单无页面层级，是模态弹出控件。触发路径：

```
页面右键 → 识别点击目标 → 弹出对应 NSMenu → 用户选择 → 执行操作 → 菜单关闭
```

## 3. 菜单结构设计

### 3.1 可编辑区域菜单 (kEditable, 优先级 0)

```
┌─────────────────────┐
│ 剪切          ⌘X    │
│ 复制          ⌘C    │
│ 粘贴          ⌘V    │
│─────────────────────│
│ 全选          ⌘A    │
└─────────────────────┘
```

**状态说明**:
- 无选中文本时："剪切"和"复制"置灰
- 剪贴板为空时："粘贴"置灰

### 3.2 链接菜单 (kLink, 优先级 1)

```
┌──────────────────────────┐
│ 在新标签页中打开           │
│ 复制链接地址               │
└──────────────────────────┘
```

**状态说明**:
- `link_url` 为空时不显示此菜单（回退到下一优先级类型）

### 3.3 图片菜单 (kImage, 优先级 2)

```
┌──────────────────────────┐
│ 将图片存储到「下载」       │
│──────────────────────────│
│ 复制图片                   │
│ 复制图片地址               │
└──────────────────────────┘
```

**状态说明**:
- `src_url` 为空时不显示此菜单
- "将图片存储到下载"是写操作，与复制操作用分隔线分组
- CORS 限制下"复制图片"点击后降级为复制图片 URL

### 3.4 选中文本菜单 (kSelection, 优先级 3)

```
┌──────────────────────────┐
│ 复制            ⌘C       │
│ 搜索"选中文本"            │
└──────────────────────────┘
```

**搜索项显示规则**:
- 选中文本 ≤ 20 字符：显示 `搜索"<全文>"`
- 选中文本 > 20 字符：显示 `搜索"<前17字符>..."`

### 3.5 空白区域菜单 (kPage, 优先级 4)

```
┌──────────────────────────┐
│ 后退            ⌘[       │
│ 前进            ⌘]       │
│ 重新加载        ⌘R       │
│──────────────────────────│
│ 查看页面源代码  ⌘U       │  ← P1 阶段启用
└──────────────────────────┘
```

**状态说明**:
- 无后退历史时："后退"置灰
- 无前进历史时："前进"置灰
- "查看页面源代码"为 P1 功能，P0 首版不显示此项；P1 阶段添加时显示在分隔线下方

## 4. 交互设计

### 菜单定位
- NSMenu 弹出位置 = 右键点击坐标（WebContents 坐标系 → NSWindow 坐标系转换）
- 菜单超出屏幕边界时由系统自动调整位置

### 菜单生命周期
1. 右键点击 → 构建 NSMenu → `popUp(positioning:at:in:)` 显示
2. 用户选择菜单项 → 执行操作 → 菜单自动关闭
3. 用户点击菜单外 → 菜单自动关闭（无操作执行）
4. 页面导航 → menu_id 失效 → 即使有缓存的菜单引用也不执行操作

### 无障碍
- NSMenu 原生支持 VoiceOver：菜单项自动可朗读
- 键盘导航：↑↓ 选择、Enter 确认、Esc 关闭（系统内建）
- 快捷键提示在菜单项右侧显示（NSMenu keyEquivalent）

## 5. 设计决策记录

| 决策 | 选择 | 替代方案 | 原因 |
|------|------|---------|------|
| 控件类型 | NSMenu（原生） | SwiftUI Menu / 自定义视图 | macOS 右键菜单必须用 NSMenu 才能正确响应右键事件和系统行为 |
| 菜单分组 | 使用分隔线区分功能组 | 无分隔线 | 遵循 macOS HIG，功能组间用 separator 提高可读性 |
| 复合场景 | 首版按最高优先级类型显示 | 合并多类型菜单项 | 降低首版复杂度，P2 迭代再合并 |

## 6. 原型说明

**本功能不需要 HTML 原型**。原因：
- NSMenu 是系统原生控件，无法用 HTML/CSS 精确模拟其外观和行为
- 菜单外观完全由 macOS 系统主题决定，无 design token 可配置
- 上方的 ASCII 菜单结构图已足够表达信息架构

视觉验证将在开发完成后通过实际运行 + 截图进行（ui-verify 阶段）。
